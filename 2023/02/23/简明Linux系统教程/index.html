由于进程之间的资源是相互独立的，所以呢，它们之间的通信没有线程那么简单。所以的通信的问题要有通过管道等方式解决。

管道(pipe)分为两种有名管道无名管道

把管道想象成你家和自来水公司的输送水的过程，是一种单项数据的传输。

管道都是有两个端口的，一个端口写入数据，一个端口读取数据。

因为通常情况下自来水公司是往你家供水的，所以类比对应的自来水公司只有写端口，你家只有读端口。

pipefd[0] refers to the read end of the pipe. pipefd[1] refers to the write end of the pipe.

任务之间的同步

管道、共享内存、消息队列这三种任务之间的通信方式，实际上是完成数据的传输的。但是有些时候，任务之间是需要完成同步的操作。同时只有一个资源，并且同时这个资源只能同时被一个资源占有。这种情况下就叫做任务之间的同步。

信号量

信号量本质上就是一个由内核维护的整数，这个整数被限定为大于等于零的整数值。

父进程每隔一定的周期消耗一个信号量，子进程每隔一定的周期发布一个信号量。

信号量在非亲缘关系的进程之间进行通信只要应用的是sem_open()这个函数。

通过信号量在线程之间进行通信。
